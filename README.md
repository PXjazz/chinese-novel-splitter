# Chinese Novel Splitter - 中文小说智能切片器

一个用于检测和切分超长小说 txt 文件的 Claude Code Skill。当小说超过 70 万 token 时，自动按章节/段落边界智能切片，输出多个分片文件，确保每个分片在 DeepSeek 等大模型的上下文窗口安全范围内。

## 📖 背景

DeepSeek 等大语言模型虽然拥有 100 万 token 的上下文窗口，但实际可用空间需要预留：
- 系统提示词
- 用户提问指令
- 模型回复内容
- 对话历史累积

因此 **安全上限设定为 70 万 token/片**，为对话留出充足空间。

## ✨ 功能特性

- **自动检测**: 读取 txt 文件并估算总 token 数，判断是否需要切片
- **智能切分**: 优先在章节标题、段落边界处切分，避免截断句子
- **上下文重叠**: 每个分片末尾保留 200 字与前一片重叠，防止信息断裂
- **多章节格式支持**: 自动识别中英文章节标记
  - `第X章`、`Chapter X`
  - `一、`、`1.` 等序号格式
- **清晰报告**: 输出每片的 token 数、字符数、文件路径

## 📐 Token 估算规则

| 字符类型 | Token 系数 | 示例 |
|---------|-----------|------|
| 中文字符 | 1.5 token/字 | `你好` ≈ 3 token |
| 英文/数字 | 0.3 token/字符 | `Hello` ≈ 1.5 token |
| 标点/其他 | 0.5 token/字符 | `，。！` ≈ 1 token |

> ⚠️ 注意：这是启发式估算，与 DeepSeek 实际 tokenizer 有 5-10% 偏差。设定 70 万上限（非 100 万）即为留出安全冗余。
